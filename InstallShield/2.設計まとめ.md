# C#アプリ（Standard / NewApp）分離対応 設計・実装・インストーラーまとめ

---

# 0. 概要

本ドキュメントは、既存C#システムに対して

- ① 通常版（Standard）
- ② Secure版（NewApp）

を同一PCで共存可能にするための設計および実装・インストーラー対応をまとめたものである。

---

# 1. 前提・方針

## 1.1 基本方針

- 製品名は共通：MyApp
- 内部識別は edition で管理
- 共存可能にする
- ①は変更不可
- ②のみ改修対象

## 1.2 Edition定義

Standard → STD  
NewApp   → NewApp

---

# 2. 対応全体像

## 2.1 対応領域

| 領域 | 内容 |
|------|------|
| 通信 | editionによる接続制御 |
| アプリ | edition送信・判定 |
| インストーラー | 共存可能にするための分離 |
| 設定 | 保存先・レジストリ分離 |
| 試験 | 組み合わせ検証 |

---

# 3. 通信設計

## 3.1 目的

- ①⇔②の誤接続防止

## 3.2 方法

### クライアント（②）

SOAP Header に edition を付与

X-MyApp-Edition: NewApp

または

<Edition>NewApp</Edition>

---

### サーバー（②）

if (request.Edition != "NewApp")
{
    throw new Exception("このサーバーはNewApp版専用です");
}

---

## 3.3 接続確認メソッド

GetCompatibilityInfo()

戻り値例:

{
  "product": "MyApp",
  "edition": "NewApp",
  "protocolVersion": 1
}

---

## 3.4 通信結果

| 通信 | 結果 |
|------|------|
| ①→① | OK |
| ②→② | OK |
| ①→② | NG |
| ②→① | NG |

---

# 4. インストーラー設計

## 分離必須項目

- ProductCode
- UpgradeCode
- インストールフォルダ
- ショートカット名
- レジストリ（クライアント）
- 設定保存先
- アンインストール表示名

---

# 5. 設定・保存先設計
